3O|||SHEET is a lightweight development environment for programming industrial controllers LogicGather, with support for LD and ST programming languages according to the IEC 61131-3 standard.

3o|||sheets IDE is a lightweight, cross-platform development environment built on OpenJDK, capable of running on single-board computers and embedded devices with minimal hardware resources.

It natively supports LD, FBD, and ST programming languages, as well as a low-level virtual machine instruction language for advanced control logic design.

In debug mode, 3osheets allows developers to pause, edit, and rewrite individual tasks in real time, without interrupting the execution of other processes — a powerful feature for live system tuning and rapid development.

The runtime supports two independent multitasking architectures:

• Classic PLC mode — a deterministic cyclic model (OB, OB100), ensuring compatibility and simplicity.
• RTOS mode — a true preemptive multitasking kernel based on a 1 ms hardware timer interrupt, enabling full task context saving and restoration for precise real-time performance.

Designed for efficiency, portability, and real-time control, 3osheets bridges traditional PLC logic with modern embedded RTOS flexibility.

3osheet Compiler

3o|||sheet Compiler is a standalone application that compiles instruction text into bytecode. It features its own programming language, .3osheet, built on a hybrid architecture — combining high-level syntax for arrays and deeply nested structures with atomic, assembly-like instructions compiled into bytecode executed by the virtual machine runtime.

The compiler supports a wide range of instructions, including virtual stack and context management operations, enabling developers to build complex multitasking programs with recursive calls and deep function transitions.

Register-based Virtual Machine

A register-based virtual machine that executes programs entirely within a safe, sandboxed environment. The 3osheet runtime can run on devices with as little as 8 KB of RAM, while providing full control over microcontroller peripherals.

It contains no low-level, architecture-dependent code, which means the runtime can be ported quickly to any microcontroller platform when needed.

Highlights
• Runs on constrained devices (≈ 8 KB RAM)
• Safe, sandboxed execution model
• Full peripheral control (GPIO, timers, UART, I²C, SPI, etc.)
• Architecture-agnostic implementation — easy porting to new MCUs
