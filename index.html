3o|||sheet Automation

Open-source PLC toolchain and lightweight virtual machine for resource-constrained microcontrollers.

Designed as a small, portable alternative to systems such as CODESYS. Runs on devices with as little as 8 KB RAM and provides deterministic execution, sandboxing, and optional fault-tolerant runtime replication.

  • ✔ IEC 61131-3 languages (LD / FBD / ST)
  • ✔ Bytecode VM for small MCUs
  • ✔ Deterministic timing
  • ✔ Runs without custom hardware
  • ✔ Optional fault-tolerant execution (DME)

System Architecture

IDE → Compiler → Bytecode → Runtime VM → Microcontroller

The project consists of three independent parts:

  • IDE — editing, debugging, deployment
  • Compiler — IEC languages → compact bytecode
  • Runtime VM — sandboxed execution on MCU

IDE

Cross-platform development environment built on OpenJDK. Runs on low-power PCs and single-board computers.

  • LD, FBD, ST editors
  • Online debugging
  • Live variable inspection
  • Program updates without PLC restart
  • Classic cyclic mode or RTOS-style preemptive mode (1 ms tick)

Compiler

  • Compiles LD / FBD / ST → compact bytecode
  • Standalone CLI tool
  • Integrates with external IDEs (Visual Studio, scripts, CI)
  • Virtual stack + context switching
  • Supports recursion and coroutines

Runtime Virtual Machine

Register-based bytecode interpreter optimized for very small microcontrollers. Designed for deterministic behavior and minimal memory usage.

  • Works on ~8 KB RAM devices
  • Sandboxed execution
  • Direct GPIO, UART, SPI, I²C, timer access
  • No dynamic allocation required
  • Architecture-independent (ARM Cortex-M, RISC-V, etc.)
  • Multiple isolated VMs on one MCU

Fault-Tolerant Execution (DME)

Divergent Multi-Version Execution (DME) is an optional runtime mode for safety-critical systems. It runs multiple replicas of the same program with decorrelated address layouts and compares their architectural state after every instruction.

  • Address-space decorrelation between replicas
  • Per-instruction state hashing
  • Single-instruction detection latency
  • Deterministic overhead

Detects replica-divergent faults such as:

  • Control-flow corruption
  • Computation errors
  • Memory corruption

Does not protect against fully correlated common-mode faults (shared clock/power/systematic hardware bugs).
